# Demo Readiness Guide

This checklist assists teams in preparing their final competition presentations.

## Presentation Structure
1. **Intake Analysis**: Describe constraints, data profile, and rubrics.
2. **Dynamic Workspace**: Show how capability is configured and run.
3. **Traceability**: Audit log demonstration.
4. **Evaluation**: Render scoring metrics.

## Dry-Run Checklist
- Verify no API keys are hardcoded.
- Confirm uvicorn starts cleanly.
- Verify uploaded mock files can be processed.
