2. Luồng LangGraph bên trong

Entry

  • analyze: đọc query, profile, chat history.
  • Heuristic nhanh: chào hỏi, chủ đề học thuật, concept đang chọn.
  • Nếu chưa rõ, gọi agent.intent_classify.

Branch

  • general: đi sang respond_general, không dùng RAG.
  • academic: chạy RAG rồi sang respond_academic.
  • academic_integrity_risk: ưu tiên nhánh an toàn.

Safety Loop

  • respond_academic: tạo câu trả lời Socratic.
  • pedagogical_reflection: kiểm code leak/đáp án trực tiếp.
  • Nếu vi phạm: gửi feedback và viết lại, tối đa 2 lần.

Điểm mấu chốt: luồng vẫn stream token về UI trong khi graph đang chạy, nên người học thấy phản hồi dần thay vì phải chờ toàn bộ pipeline kết thúc.

3. Mapping với Braintrust spans

Span Đại diện cho đoạn nào Dùng để xem gì
chat.stream Toàn bộ request stream từ backend. Latency tổng, lỗi tổng, bottleneck request.
LangGraph Thời gian chạy graph agent. Orchestration có chậm không.
agent.intent_classify LLM phân loại intent khi heuristic chưa đủ. Chi phí ẩn trước khi trả lời.
rag.retrieve Truy xuất slide/context liên quan. RAG, embedding, vector RPC có chậm không.
respond_academic Node tạo phản hồi học thuật. Prompt, LLM, citation validation.
llm.respond_stream LLM sinh câu trả lời dạng stream. First token, total generation time.
pedagogical_reflection Critic kiểm định Socratic khi có rủi ro. Guardrail có làm tăng latency không.
chat.message_save Lưu lịch sử chat vào DB. Persistence có lỗi/chậm không.
